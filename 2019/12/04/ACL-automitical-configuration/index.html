Primitive Design: check

如果两个数据包被阻止或被允许通过的情况相同，我们认为这两个数据包属于同一个 forwarding equivalence class (FEC)，这时 (1) 返回 true

$[1]{FEC}={1},\ [2]{FEC}={2,3},\ [4]{FEC}={4},\ [5]{FEC}={5,6},\ [7]_{FEC}={7}$

对于每组 FEC，即 $[hi]{FEC}$：

  • 遍历所有路径，将允许 $[hi]{FEC}$ 通过的路径 $p_i$ 加入 $y$

  • 在进行 ACL 更新后，对于 $y$ 中的路径，如果该路径上有节点通过状态和之前不一样，那么就说明更新前后数据包到达不一致，(2) 返回 true

Example: 考虑 $[2]{FEC}={2,3}$，有两条路径，记作 $p_0=$ 和 $p_2=$。当 “deny dst 2.0.0.0/8” 从 $D_2$ 移到 $A_1$，此时，$c{p0}=True$，$c’_{p0}=False$，即，更新违反了一致性。

Primitive Design: fix

Check 原语找到一个违反一致性的原因就返回了，现在需要找到违反该同一一致性的原因，并通过 SMT Solver 生成修复规则，迭代，直到没有任何地方违反一致性。
